# 技術選定ドキュメント（初学者向け）

**Purpose**
このドキュメントは、マンガ所持管理アプリ（Web版）を作るための技術選定ドキュメント

**Policy**
1. Webで動くものを、なるべく早く作れること
2. 後から機能追加しやすいこと
3. 初心者でも学びやすいこと

**Decisions**
1. **フロントエンド: Next.js + React + TypeScript**
理由: Webアプリを作るための定番の組み合わせで、情報が多く学びやすい。画面づくりが早い。
2. **バックエンド: FastAPI（Python）**
理由: PythonでAPIを作るための軽量フレームワーク。シンプルで分かりやすい。
3. **データベース: PostgreSQL**
理由: 本やシリーズなどのデータをしっかり保存でき、将来の拡張にも強い。
4. **ORM・マイグレーション: SQLAlchemy + Alembic**
理由: PythonからDBを扱いやすくし、DBの変更履歴を管理できる。
5. **認証: Google OAuth + JWT**
理由: Googleでログインできるようにして、本人確認を簡単にする。
6. **バックアップ: Google Drive API**
理由: 要件にあるGoogle Driveへのバックアップを実現できる。
7. **非同期処理: Celery + Redis（必要になったら）**
理由: 新刊チェックなど、時間のかかる処理を裏側で動かせる。
8. **インフラ**
理由: まずは簡単に公開できる場所を使う。

**Infra Choices**
1. **フロントエンド: Vercel / Netlify**
説明: Next.jsを簡単に公開できる。
2. **バックエンド: Render / Fly.io / Cloud Run**
説明: 小さく始めて、必要に応じて拡張できる。

**Glossary**
1. **API**
説明: 画面とサーバーがデータをやり取りするための窓口。
2. **データベース**
説明: 本の情報を保存する場所。
3. **ORM**
説明: Pythonのコードでデータベースを扱いやすくする仕組み。
4. **マイグレーション**
説明: データベースの設計変更を履歴として管理する仕組み。
5. **JWT**
説明: ログイン済みかどうかを確認するための短いトークン。
6. **PWA**
説明: Webアプリをスマホでアプリっぽく使えるようにする仕組み。

**Scope**
1. Next.js Webアプリ
2. FastAPI サーバー
3. PostgreSQL
4. Googleログイン
5. Google Driveバックアップ（手動）

**Open Items**
1. フロントエンドのデプロイ先を確定する（Vercel / Netlify）
2. バックエンドのデプロイ先を確定する（Render / Fly.io / Cloud Run）
3. 通知機能の要否と方式を決める
4. Drive連携を手動にするか自動にするか決める
5. 端末間同期が必要か決める
